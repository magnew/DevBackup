Ideal Solution

This page is a mockup of the scenario we are shooting for - customer styles are preserved and the Splunk table gets the nice Bubbles styles. To achieve this, Splunk styles would not have the same names as the standard Bootstrap names. For example navbar would be renamed to spl-navbar.

.spl-navbar {
  overflow: visible;
  margin-bottom: 16px;
  *position: relative;
  *z-index: 2;
}

This way our styles would effectively be namespaced and thus only applied to controls where we specifically intend them to be. With this solution a standard Bootstrap navbar (or table or anything else) could be included in the page by using the standard class names, but the page could also include Splunk-style controls by using the explicit Splunk style names. So to get a Splunk navbar instead of a standard navbar markup would look like this:

  <div class="spl-navbar">
    <div class="spl-navbar-inner">
      ...

Our controls - such as the table - would use the Splunk styles explicitly, allowing them to exist in the page side-by-side with elements styled with standard Bootstrap.
